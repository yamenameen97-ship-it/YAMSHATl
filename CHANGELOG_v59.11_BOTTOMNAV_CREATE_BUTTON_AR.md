# تحديث v59.11 — توحيد زر الإنشاء (+) في الهيدر السفلي

## المشاكل التي عُولجت

1. **زر (+) كان مخبوطاً ويختفي جزئياً** في بعض الصفحات (يظهر مقصوصاً من الأعلى أو الأسفل)، خصوصاً بسبب ارتفاعه الزائد (48×48) وخروجه عن حدود الهيدر.
2. **عدم التناسق مع جيرانه**: كان الزر بارزاً وكبيراً ومختلف الشكل تماماً عن بقية أزرار الهيدر (حسابي / الريلز / الدردشات / الرئيسية).
3. **النص أحياناً لا يظهر** بسبب القص العمودي.

## التغييرات

### `frontend/src/components/mobile/BottomNav.jsx`

- إعادة هيكلة زر (+) ليصبح بنفس بنية بقية الأزرار:
  - أيقونة (+) صغيرة بالأعلى (18×18) داخل خلفية بنفسجية بحواف دائرية ممدودة (pill).
  - نص توصيفي بالأسفل بنفس حجم وأسلوب بقية الأزرار.
- النص يتغيّر ديناميكياً حسب الصفحة الحالية:
  | الصفحة | النص |
  |--------|------|
  | الرئيسية (`/`) | `منشور جديد` |
  | المجموعات (`/groups`) | `إنشاء مجموعة` |
  | الريلز (`/reels`) | `ريلز جديد` |
  | الدردشات (`/inbox` أو `/chat`) | `دردشة جديدة` |
  | الستوري (`/stories`) | `ستوري جديد` |
- تقليل ارتفاع الهيدر الكلي إلى `64px` (بدلاً من `72px`) لأن الزر لم يعد بارزاً.
- توحيد `padding` و `gap` بين كل العناصر لضمان محاذاة بصرية كاملة.
- ضبط `direction: rtl` على المستوى الجذر و `flex-direction: row-reverse` للحاوية الداخلية حتى يكون الترتيب من اليمين إلى اليسار: الرئيسية → الدردشات → (+) → الريلز → حسابي.
- استجابة كاملة لجميع الشاشات (320px → 1024px+) بما في ذلك Redmi Note 8 (393px).

## النتيجة

- زر (+) أصبح ظاهراً بالكامل بدون أي قص في جميع الصفحات.
- متناسق تماماً مع الأزرار من اليمين واليسار (نفس الحجم، نفس بنية الأيقونة + النص).
- وظيفته تتغير ديناميكياً حسب السياق (منشور / ريلز / دردشة / ستوري / مجموعة).

# سجل التغييرات — v59.13.33 — إصلاح أزرار الإعدادات الرئيسية

## المشكلة المُبلَّغة
> «أزرار الإعدادات الرئيسية موجودة، لكن عند الضغط على أي زر ما يعطيني شي وما يفتح الخدمة الخاصة به.»

كان معظم الأزرار داخل لوحات الإعدادات (الحساب، المصادقة الثنائية، التطبيقات المرتبطة، تنزيل البيانات، المساعدة، الملاحظات، عن التطبيق، القانوني) **يفتقد إلى `onClick` تمامًا** — أزرار JSX خام بدون أي معالج. الضغط عليها لا يُحدث شيئًا.

## ما تم إصلاحه (الملف: `frontend/src/pages/Settings.jsx`)

### 1. تبويب «الحساب»
- زر **تعديل البريد** → مودال يطلب البريد الجديد + إرسال رابط تأكيد (محفوظ في `prefs`).
- زر **تعديل رقم الهاتف** → مودال + رمز تحقق SMS.
- زر **تغيير كلمة المرور** → مودال 3 حقول (الحالية/الجديدة/التأكيد) + تحقق طول وتطابق.
- زر **تعديل اسم المستخدم** → مودال إدخال.
- زر **تعديل تاريخ الميلاد** → مودال `<input type="date">`.
- زر **تعديل الدولة** → مودال `<select>` بقائمة 18 دولة عربية + أخرى.
- زر **إيقاف الحساب مؤقتًا** → مودال تأكيد + سبب اختياري + تسجيل خروج بعد التأكيد.
- زر **حذف الحساب نهائيًا** → مودال يتطلب كتابة «حذف نهائي» نصًا للتأكيد.
- زر **تحويل لحساب أعمال** → مودال (فئة العمل + موقع إلكتروني).

### 2. تبويب «المصادقة الثنائية»
- **إعداد تطبيق المصادقة** → مودال إدخال رمز 6 أرقام + تحقق Regex.
- **رمز عبر البريد** + **رمز عبر SMS** → الآن `SettingsToggle` يعمل ويُحفظ.
- **إضافة مفتاح أمان** → مودال تسمية المفتاح.
- **عرض/توليد رموز الاسترداد** → يفتح نافذة بـ 10 رموز عشوائية + زر «نسخ الكل» للحافظة.
- **البصمة/Face ID** → toggle حقيقي مربوط بـ prefs.

### 3. تبويب «الجلسات»
- زر **إنهاء كل الجلسات عدا الحالية** → كان بلا handler. الآن: تأكيد + `Promise.all` لإنهاء كل الجلسات + إعادة جلب القائمة.

### 4. تبويب «التطبيقات المرتبطة»
- أزرار **ربط / إلغاء الربط** لـ Google/Apple/Facebook/X → الآن toggle حالة فعلي مع تأكيد قبل فك الربط، والوصف يتحدث ديناميكيًا (مرتبط/غير مرتبط).

### 5. تبويب «البيانات والتخزين»
- زر **مسح الوسائط المنزّلة** → مسح فعلي لـ localStorage entries + Cache API.

### 6. تبويب «تنزيل بياناتي»
- 3 أزرار «طلب» (كامل/نشاط/وسائط) → كل واحد يميّز نوع الطلب في رسالة التأكيد.

### 7. تبويب «المساعدة»
- **تواصل مع الدعم** → مودال (موضوع + رسالة).
- **الأسئلة الشائعة** → نافذة معلومات داخلية بـ 6 أسئلة وأجوبة.
- **الإبلاغ عن مشكلة** → مودال (نوع المشكلة + تفاصيل).
- **دروس البدء السريع** → نافذة بـ 6 دروس قابلة للنقر.

### 8. تبويب «إرسال ملاحظات»
- **قيّم التطبيق** → مودال (نجوم 1-5 + تعليق).
- **اقترح ميزة** → مودال (عنوان + تفاصيل).
- **بلّغ عن خطأ** → نفس مودال البلاغ.

### 9. تبويب «عن التطبيق»
- **ما الجديد** → نافذة تعرض changelog v59.13.33 و v59.13.32.
- **الموقع الرسمي** → نافذة معلومات.

### 10. تبويب «القانوني»
- **سياسة Cookies** / **DMCA** / **إرشادات المجتمع** → 3 نوافذ معلوماتية بنصوص كاملة.
- شروط الاستخدام و سياسة الخصوصية تبقى عبر `navigate('/terms')` و `navigate('/privacy')` (موجودة بالفعل في `App.jsx`).

## البنية التحتية المضافة
- مكوّن `SettingsModal` موحّد: يدعم حقول `text/email/tel/password/date/url/textarea/select` + زر تأكيد/إلغاء + حالة danger.
- مكوّن `InfoModal` لعرض محتوى تعليمي/قانوني.
- CSS modal احترافي: backdrop-blur + animations + responsive + RTL.
- جميع البيانات الجديدة تُحفظ في `localStorage` تحت `yamshat:app-prefs` ولا تُفقد عند إعادة التحميل.

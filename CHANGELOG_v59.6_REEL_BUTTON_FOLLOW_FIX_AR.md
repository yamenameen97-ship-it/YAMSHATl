# v59.6 — إصلاح زر "ريلز جديد" + زر المتابعة في الريلز

## 1) زر "ريلز جديد" / "ستوري جديد" في الـ BottomNav

**الملف المعدّل:** `frontend/src/components/mobile/BottomNav.jsx`

### قبل
- زر مركزي صغير (40×40px) بداخله علامة `+` فقط، مدمج بالكامل ضمن الشريط السفلي.
- نفس الشكل في جميع الصفحات بدون تمييز للريلز.

### بعد
- عند الدخول إلى `/reels` أو `/stories`:
  - الزر **يتحوّل لمستطيل ريلز/Clapper** (مطابق للصورة المرجعية الأولى).
  - **الحجم 56×56px** مع `border-radius: 16px`.
  - **يرتفع للأعلى** بمقدار `margin-top: -22px` ليظهر "طافياً" فوق النافبار.
  - **إطار بلون النافبار** (`box-shadow: 0 0 0 4px #0A0D1A`) لإبرازه.
  - **توهج بنفسجي** قوي تحت الزر.
  - لصاقة "ريلز جديد" تظهر تحته بلون بنفسجي فاتح وخط عريض (700).
- في باقي الصفحات يبقى زر `+` التقليدي كما كان.

### التطبيق الديناميكي
- تمت إضافة `iconKind` إلى `resolveCreateAction(pathname)` لتمييز نوع الزر:
  - `reel` / `story` → شكل clapper + يطفو.
  - `post` / `group` / `chat` → شكل `+` العادي.

## 2) إصلاح زر المتابعة (+) في الـ Reels

**الملف المعدّل:** `frontend/src/pages/Reels.jsx`

### المشكلة
- عند الضغط على زر المتابعة، تظهر علامة ✓ لجزء من الثانية ثم تعود لـ `+`.
- السبب: الفرونت يستدعي `POST /users/<username>/follow` و `DELETE /users/<username>/follow` — وهذه المسارات **غير موجودة في الباك إند**.
- نتيجة فشل الطلب: يدخل `catch` ويعمل **rollback** للحالة الأصلية → الزر يعود لشكله السابق.

### الحل
تم توحيد الاستدعاء على **endpoint موجود فعلياً** في الباك إند:
```js
POST /api/users/follow
Body: { following: "<username>" }
```
هذا المسار (`users.py` → `follow_or_unfollow`) يعمل **toggle تلقائياً** ويعيد:
```json
{ "username": "...", "following": true|false, "followers": 0, "following_count": 0 }
```

### Fallback احتياطي
في حال فشل الاستدعاء الأساسي، يجرب المسار البديل:
```js
POST /follow/<username>     // (متابعة)
POST /unfollow/<username>   // (إلغاء)
```
المتاح في `profile_v2.py` (Flask Blueprint).

### تحسينات إضافية
- مزامنة حقيقية مع استجابة الخادم — إذا أعاد الخادم `following=false` (مثل إلغاء متابعة)، يتم تحديث الحالة بدقة بدلاً من الافتراض.
- رسائل خطأ مخصصة:
  - `401 / 403` → "يلزم تسجيل الدخول للمتابعة"
  - أخطاء أخرى → "تعذّر تحديث المتابعة"
- لا يحدث rollback إلا عند فشل **حقيقي** بعد تجربة المسارين.

## النتيجة المتوقعة
1. ✅ زر "ريلز جديد" يظهر كبيراً وطافياً فوق النافبار في صفحة الريلز، مطابقاً تماماً للصورة المرجعية الأولى.
2. ✅ زر المتابعة الوردي يعمل بشكل دائم — يتحول إلى ✓ ويبقى ✓ بعد المتابعة بنجاح.

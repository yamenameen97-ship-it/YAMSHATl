# CHANGELOG — v59.8 — إصلاح سلوك الكاميرا في صفحة الريلز
==========================================================

## 🎯 ملخّص الإصلاحات

تم تعديل صفحة `ReelComposer.jsx` بحيث:

1. **لا تُفتح الكاميرا تلقائياً** عند فتح صفحة رفع الريلز.
   - سابقاً: كانت `getUserMedia` تُستدعى فور تركيب المكوّن → تشغيل الكاميرا فوراً.
   - الآن: الكاميرا تبقى مغلقة وتظهر شاشة بديلة عليها زر "اضغط لفتح الكاميرا".

2. **إضافة زر أيقونة الكاميرا في الشريط العلوي** (بجانب كبسولة "إضافة صوت 🎵").
   - عند الضغط: تُفتح الكاميرا (`cameraOn=true` ويُستدعى `getUserMedia`).
   - عند الضغط مرة أخرى: تُغلق الكاميرا فوراً (إيقاف كل tracks).
   - الزر يحتوي على حالتين بصريتين: عادية (شفافة) ومُفعّلة (تدرج بنفسجي مع توهج).

3. **زر التسجيل البنفسجي الأوسط أصبح ذكياً**:
   - الكاميرا مغلقة → يطلب فتح الكاميرا أولاً.
   - الكاميرا مفتوحة → يبدأ/يوقف التسجيل كالمعتاد.
   - معاينة فيديو من المعرض → يتحوّل إلى زر "تأكيد ونشر" أخضر.

4. **عند اختيار مقطع من المعرض/الهاتف**:
   - يُعرض الفيديو في وضع **معاينة كاملة** (autoplay + loop + muted) ملء الشاشة.
   - شريط أدوات التحسينات يبقى نشطاً لتجربتها لايف: **السرعة / الفلاتر / المؤثرات / تحسين / تجميل / التخطيط / الجودة / كتم / الترجمة**.
   - تُخفى الأدوات غير ذات الصلة في وضع المعاينة (المدة، المؤقت، الفلاش، قلب الكاميرا، الميكروفون، فلاتر الضوضاء) لأنها مرتبطة بالتسجيل المباشر فقط.
   - تظهر شارة علوية: **"وضع المعاينة — جرّب الفلاتر والمؤثرات والسرعة قبل النشر"**.
   - الفيديو يستجيب فوراً لأي تغيير في الفلتر/التجميل (CSS filter) وفي السرعة (`playbackRate`).

5. **تحسينات إضافية**:
   - عند اختيار ملف من المعرض → تُغلق الكاميرا تلقائياً (توفير موارد + خصوصية).
   - عند فتح الكاميرا → تُمسح معاينة المعرض.
   - تنظيف صحيح لـ `URL.createObjectURL` عند الخروج/التبديل (لا تسريب ذاكرة).
   - رسائل خطأ واضحة عند رفض إذن الكاميرا (بدلاً من فشل صامت).
   - زر "تأكيد" السفلي يُعطّل تلقائياً عند عدم وجود فيديو جاهز للنشر.

## 📁 الملفات المعدّلة

## 🧪 سيناريوهات الاختبار

| السيناريو | السلوك المتوقع |
|---|---|
| فتح `/reels/new` لأول مرة | شاشة سوداء عليها زر "اضغط لفتح الكاميرا" + رابط للمعرض. لا طلب صلاحية كاميرا. |
| الضغط على زر الكاميرا (أعلى) | يُطلب إذن الكاميرا → تبدأ المعاينة الحية. |
| الضغط على زر الكاميرا مرة أخرى | الكاميرا تُغلق فوراً، تعود الشاشة البديلة. |
| الضغط على زر التسجيل الأوسط والكاميرا مغلقة | يفتح الكاميرا (نفس سلوك زر الكاميرا). |
| اختيار فيديو من "المعرض" | الكاميرا تُغلق، يبدأ تشغيل الفيديو في وضع معاينة. شارة بنفسجية تظهر أعلى. |
| تغيير الفلتر/التجميل أثناء المعاينة | يُطبَّق فوراً على الفيديو (CSS filter). |
| تغيير السرعة أثناء المعاينة | الفيديو يتسارع/يبطئ فوراً (`playbackRate`). |
| الضغط على X (إلغاء) | تُمسح المعاينة/التسجيل وتُغلق الكاميرا. |
| الضغط على ✓ (تأكيد) | يبدأ الرفع للملف الحالي (مسجّل أو من المعرض). |

## 🔒 الخصوصية والأداء

- لا يتم طلب صلاحية الكاميرا/الميكروفون إلا بتفاعل صريح من المستخدم → مطابق لسياسات المتصفحات الحديثة وتوصيات Apple/Google.
- الكاميرا تُغلق تلقائياً عند مغادرة الصفحة أو اختيار ملف من المعرض → تخفيض استهلاك البطارية والشبكة على الهواتف منخفضة الإمكانيات.

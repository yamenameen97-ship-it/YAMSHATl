# إصلاحات v25 — تاريخ النشر اللحظي + زر "العودة للأعلى" (فرونت إند فقط)

## 1) إصلاح تاريخ النشر ليكون لحظياً ودقيقاً ✅

### الجذر الفني للمشكلة
- الـ Backend يخزّن `created_at` و `published_at` باستخدام `datetime.utcnow()` (UTC) — وحقول `DateTime` في SQLAlchemy **بدون `timezone=True`**.
- عند تسليم القيمة عبر JSON تخرج بصيغة `"2026-06-12T23:15:00"` **بدون لاحقة `Z`** ولا offset.
- المتصفح عند تنفيذ `new Date("2026-06-12T23:15:00")` يفسّرها **توقيتاً محلياً** (وفق ECMA-262)، فتصبح القيمة الناتجة لمن يعيش في `+03:00` (السعودية/اليمن/الأردن/مصر/الكويت…) **متقدمة 3 ساعات** عن الحقيقة.
- النتيجة: منشور تم نشره الآن يظهر فوراً "منذ 3 ساعة" — وهي بالضبط الصورة التي أرفقتها.

### تأكيد عملي (تم في الاختبار)
```
Backend-style timestamp (no Z): 2026-06-13T02:20:24.549
[قبل الإصلاح] الفرق بحساب JS الساذج: 10800 ثانية ≈ 3 ساعات ❌
[بعد الإصلاح] الفرق الصحيح:                   0 ثانية → "الآن" ✓
```

### الحل المنفّذ
أُنشئ مرفق موحّد:

**`frontend/src/utils/timeFormat.js`** يقدّم:
- `parseServerDate(value)` — يضيف `Z` تلقائياً للتواريخ ISO القادمة من الباك إند عند غياب TZ، فيُفسَّر التاريخ بشكل صحيح كـ UTC.
- `timeAgoAr(value)` — يحسب الفرق بـ `Date.now()` (الذي يعكس توقيت جهاز المستخدم بدقة الميلي ثانية)، مع نصوص عربية متدرّجة:
  `الآن → منذ لحظات → منذ دقيقة/X دقيقة → منذ ساعة/X ساعة → أمس/X يوم → X أسابيع → X شهر → X سنة`.
- يحتوي على عتبة "الآن" لأول 30 ثانية، ويتعامل مع التواريخ المستقبلية البسيطة (انحراف ساعة الخادم) دون انعكاسات سلبية.
- `formatLocalDateTimeAr(value)` — يعطي تاريخاً كاملاً بتوقيت جهاز المستخدم (يُستخدم في الـ tooltip عند المرور بالماوس).

### تكامل لحظي مع الواجهة
- داخل كل بطاقة منشور يجري `setInterval` بـ **30 ثانية** يعيد حساب «منذ كم» دون أي طلب شبكي ودون إعادة تحميل، فيتحوّل "منذ دقيقة" إلى "منذ دقيقتين" حياً.
- التنظيف (`clearInterval`) عند إزالة البطاقة من الـ DOM يمنع التسرب.

## 2) ميزة فرونت-إند 100% للشاشة الرئيسية ✅
### زر «العودة للأعلى» (Scroll-to-Top FAB)

**ما هو؟** زر عائم دائري يظهر بعد تمرير المستخدم لأسفل أكثر من **500 بكسل** في شاشة المنشورات الرئيسية فقط. عند النقر يُعيد السكرول إلى الأعلى بانسيابية.

**لماذا هذا الخيار؟**
- ✔ لا يحتاج Backend ولا جدول قاعدة بيانات.
- ✔ لا يحتاج تخزيناً للحالة عبر الجلسات.
- ✔ يحلّ مشكلة UX حقيقية: تصفّح الخلاصة الطويلة يجعل العودة للأعلى مرهقة، خصوصاً على الجوال.
- ✔ يتكامل مع التصميم الحالي (تدرّج بنفسجي مطابق لهوية يام شات) ويدعم RTL ووضع `prefers-reduced-motion`.

**الملف الجديد**: `frontend/src/components/feed/ScrollToTopFab.jsx` (مكوّن مستقل بـ CSS مضمّن — لا تعديل على أي ملف ستايل آخر).

**موقع التضمين**: داخل `FeedEnhanced.jsx` بعد `MainLayout` مباشرة بحيث يظهر في الشاشة الرئيسية **فقط** (الجوال + الويب).

**خصائص قابلة للضبط**:
- `threshold` (افتراضي 500px) — قيمة السكرول التي يبدأ الزر بالظهور عندها.
- `targetSelector` — يمكن استهداف عنصر سكرول داخلي بدلاً من window.

**سلوك بصري**:
- يدخل بحركة `translateY + scale` ناعمة (220ms).
- يختفي تلقائياً عند الاقتراب من الأعلى.
- `box-shadow` بنفسجي يتوهج عند المرور بالماوس.
- على سطح المكتب يستقر يساراً-أسفل بـ 28px، وعلى الجوال يرتفع إلى 96px لتفادي الشريط السفلي.

# v87.15 — إصلاحات جوال حقيقية وآمنة للنشر

هذا التحديث لا يعتمد فقط على CSS هشّ أو على `:has()`، بل أضاف **وسم صفحة فعلي** داخل `MainLayout` + تعديلات مباشرة داخل الصفحات الحساسة، حتى تظهر الإصلاحات بعد الرفع إلى GitHub والنشر على Render بشكل واضح.

## المشاكل التي تم إصلاحها

### 1) صفحة الستوريات الرئيسية
- المشكلة: أول صف الستوريات كان يصعد تحت الهيدر/الأزرار العلوية على ويب الجوال.
- السبب العملي: بعض الإصلاحات القديمة كانت تعتمد على `:has(.yam-stories-page)` أو تتأثر بطبقات CSS أقدم تصفّر `padding-top` للحاوية الأم.
- الإصلاح:
  - إضافة `data-page="stories"` من `MainLayout`.
  - جعل CSS يستهدف صفحة الستوريات مباشرة عبر `data-page` بدل الاعتماد على `:has()` فقط.
  - إضافة `data-yam-stories-root="true"` وتحسين padding/box-sizing داخل الصفحة نفسها.

### 2) شات المجموعة — حقل الكتابة/زر الإرسال/الكيبورد
- المشكلة: تراكب في منطقة الإدخال، محاذاة RTL غير مريحة، وإحساس بوجود زر إرسال ظاهر بينما منطقة النقر الفعلية تتداخل تحته.
- السبب العملي: تراكم قواعد قديمة بين `group-chat.css` والإصلاحات العامة، مع اعتماد زائد على CSS فقط في منطقة حساسة للكيبورد.
- الإصلاح:
  - جعل footer الإدخال بتنسيق Grid ثابت مباشر داخل `GroupChat.jsx`.
  - فرض أبعاد ومحاذاة واضحة لزر الإرسال وزر الإرفاق وحقل الإدخال inline داخل المكوّن.
  - رفع `z-index` وعزل الطبقة (`isolation: isolate`) لمنع التراكب.
  - جعل مسافة أسفل الرسائل ومواضع قوائم المرفقات/شريط الرفع تتجاوب مع `--yam-keyboard-offset`.
  - عند focus على الحقل يتم دفع العرض إلى الأسفل تلقائياً ليبقى الإدخال مرئياً.

### 3) صفحة كتابة المنشور
- المشكلة: السحب لأعلى/أسفل واللمس كانا يتعطلان أو يصبحان غير موثوقين على الجوال.
- السبب العملي: الإصلاح القديم كان يعتمد على مطابقة الحاوية عبر `:has(.ympc-page)`؛ إذا لم تُطبّق القاعدة في بيئة النشر أو تم تجاوزها، يرجع السلوك المعطوب.
- الإصلاح:
  - إضافة `data-page="post-composer"` من `MainLayout`.
  - استهداف الحاوية الأم مباشرة في CSS عبر `data-page`.
  - إضافة `data-yam-post-composer="true"` وتحسين padding السفلي داخل الصفحة.
  - زيادة المساحة السفلية حتى لا تتداخل الواجهة مع العناصر الثابتة على الجوال.

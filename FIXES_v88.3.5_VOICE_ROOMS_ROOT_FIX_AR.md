# الإصلاحات v88.3.5 — إصلاح جذري لفشل إنشاء الغرف الصوتية

**التاريخ**: 2026-07-18
**الحالة**: إصلاح **جذري وشامل** (Root Cause Fix)

---

## 🎯 المشكلة

عند محاولة إنشاء غرفة صوتية داخل المجموعات (أو من صفحة الغرف الصوتية) تظهر رسالة:
> **خدمة الغرف الصوتية غير متوفرة حالياً — حاول مجدداً بعد قليل**

رغم أن الباك اند يعمل وباقي الخدمات (المنشورات، الشات، الاستطلاعات) تعمل بشكل طبيعي.

---

## 🔍 السبب الجذري (Root Cause Analysis)

بعد مراجعة الفرونت اند + الباك اند تبيّن ثلاث مشاكل متعاونة كانت تُسبّب `404 Not Found`:

### 1. `redirect_slashes=True` (السلوك الافتراضي في FastAPI)
- الفرونت اند يستدعي `/api/voice/rooms` (بدون slash).
- FastAPI الافتراضي يعيد `307 Redirect` إلى `/api/voice/rooms/` (بslash).
- عبر **cross-origin** (Render: yamshat8 → yamshat-1ya4)، **يفقد المتصفح** الكوكيز و`Authorization` header عند إعادة التوجيه في بعض إصدارات الأندرويد/PWA.
- النتيجة: الطلب يصل مجهول الهوية فيُرفض بـ `404` أو `401` غامض.

### 2. استيراد `bcrypt` الكسول داخل `create_room()`
- الملف `voice_rooms.py` كان يستورد `from passlib.hash import bcrypt` **داخل** الدالة.
- إذا كانت مكتبة `passlib` غير مثبتة على السيرفر (نشر ناقص)، الاستيراد يفشل عند أول استدعاء لغرفة **خاصة** فقط، فيُرمى `500` غير مفهوم.
- الأسوأ: يفشل بصمت في بعض المسارات.

### 3. راوتر مسجّل تحت prefix واحد فقط
- كان يُسجَّل على `/api/voice` + alias قديم `/api`.
- إذا حصل تعارض في التحميل (مثلاً import صامت فشل)، لا يوجد مسار احتياطي واضح.
- الفرونت اند القديم كان يجرّب مسارين فقط ولا يعرف كيف يُميّز بين "الراوتر مفقود" و"مشكلتك أنت".

## ✅ الإصلاحات المُنجَزة

### الباك اند

**1. `backend/app/main.py`**
- إضافة `redirect_slashes=False` لتعطيل التحويل التلقائي بين `/rooms` و `/rooms/` — لا مزيد من فقدان الكوكيز عبر cross-origin.
- تسجيل راوتر voice_rooms على **بادئة إضافية**: `/api/voice-rooms` (بجانب `/api/voice` و`/api`).

**2. `backend/app/api/routes/voice_rooms.py`**
- **استيراد `bcrypt` بشكل آمن على أعلى الملف** داخل try/except:
  - إذا فُقدت `passlib` لا يفشل الراوتر بأكمله — فقط تُعطَّل ميزة الغرف الخاصة مع تحذير في اللوج.
  - رسالة عربية واضحة للمستخدم: "خدمة الغرف الخاصة غير مهيّأة على السيرفر".
- **endpoint تشخيصي جديد** `GET /rooms/_ping` (بدون مصادقة) لتأكيد أن الراوتر محمّل. الفرونت اند يستخدمه لتمييز:
  - "الراوتر غير موجود" (المستخدم يحتاج انتظار) vs
  - "الراوتر موجود لكن طلبك فشل" (نصح المستخدم بإعادة تسجيل الدخول).
- **مضاعفة decorator** على كل endpoints الأساسية:
  ```python
  @router.post("/rooms")
  @router.post("/rooms/")
  ```
  حتى يعملان في كل الحالات دون الحاجة لـredirect.
- **تحقق مبكر واضح** مع رسائل خطأ عربية:
  - عنوان الغرفة مطلوب
  - عدد المقاعد يجب أن يكون 2..15
  - الغرفة الخاصة تتطلب كلمة مرور
- **try/except عام** حول `create_room` مع `db.rollback()` و logging تفصيلي.

### الفرونت اند

**3. `frontend/src/features/engagement/api/engagementApi.js`**
- `voiceRoomsApi` الآن يجرّب **3 بوادئ**:
  1. `/api/voice/rooms` (الرئيسي)
  2. `/api/voice-rooms/rooms` (alias جديد أوضح)
  3. `/api/rooms` (alias قديم متوافق)
- إضافة `voiceRoomsApi.ping()` — دالة تشخيصية ترجع `{ available: bool, path?: string }`.
- كل endpoints تستخدم الآن مولّد `vrPaths()` الذي يبني القائمة تلقائياً.

**4. `frontend/src/pages/VoiceRoomsPage.jsx`**
- عند فشل الإنشاء بـ `404` نستدعي `ping()` **تلقائياً** لتمييز الحالتين:
  - **الراوتر متوفر** → رسالة: *"تعذر إنشاء الغرفة — جرّب تحديث الصفحة وإعادة تسجيل الدخول"*
  - **الراوتر غير متوفر** → رسالة: *"خدمة الغرف الصوتية غير متوفرة حالياً — حاول مجدداً بعد قليل"*
- معالجة أفضل لأكواد `403`, `422`, `503` مع رسائل عربية دقيقة.

## 🧪 كيفية اختبار الإصلاح بعد النشر

### 1. تحقق أن الراوتر محمّل
افتح في المتصفح مباشرة:
```
https://YOUR-BACKEND.onrender.com/api/voice/rooms/_ping
```
يجب أن ترى:
```json
{"ok": true, "service": "voice_rooms", "bcrypt_available": true, "ts": "..."}
```

### 2. تحقق من التشخيص الشامل
```
https://YOUR-BACKEND.onrender.com/api/_diag/routers
```
تحقق أن `app.api.routes.voice_rooms.router` قيمته `"ok": true`.

### 3. اختبار الإنشاء من الواجهة
- سجّل دخول
- المجموعات → إنشاء غرفة صوتية
- أدخل عنوان → عام → 8 مقاعد → غير خاصة
- اضغط **"🚀 إنشاء وبدء الغرفة"**
- يجب أن تدخل الغرفة مباشرة بدون رسالة خطأ.

### 4. إذا ظهر خطأ بعد الإصلاح
افتح كونسول المتصفح — سترى الآن **رسالة تشخيصية دقيقة**:
```
[VoiceRoom.create] failed: { status: XXX, detail: "...", err: {...} }
```
هذه الرسالة تكشف السبب الحقيقي (401/403/422/503/network) بدل الرسالة الغامضة القديمة.

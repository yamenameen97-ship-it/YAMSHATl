# إصلاح تجربة الموبايل — YAMSHAT
## (Mobile UX Fix — Scroll, Touch, Update Banner)

## ✋ المشاكل المُبلَّغ عنها

1. **عدم القدرة على السحب لفوق/تحت** على الموبايل — الصفحة "واقفة" ولا تستجيب للمس واللمس على الواجهة الحالية.
2. **زر التحديث في الأسفل (`تحديث الآن` / `لاحقاً`)** بألوان النظام الخضراء الافتراضية، مزعج جداً، **يغطي زر تسجيل الدخول الحقيقي** ويبدو وكأنه الزر الفعلي.
3. **التجربة لا تشعر كأنها تطبيق** — استجابة لمس بطيئة، لا يوجد ارتداد طبيعي، الـ scroll ليس سلساً.

---

## 🔍 السبب الجذري

عند التحقيق، اكتشفت المشكلة بالضبط في ملف `src/styles/smooth-touch-experience.css` — كان يحتوي على القواعد التالية:

```css
body {
  position: fixed;          /* ❌ قاتل التمرير */
  width: 100%;
  height: 100vh;
  overflow: hidden;         /* ❌ يقفل أي سحب رأسي */
}

#root {
  position: fixed;          /* ❌ نفس المشكلة */
  top: 0; left: 0; right: 0; bottom: 0;
  overflow: hidden;         /* ❌ يقفل التمرير */
  ...
}
```

كانت النية أن "تُشعر التطبيق كأنه أصلي"، لكن النتيجة كانت **شلّ التمرير بالكامل** على الموبايل. الصفحة كانت مثبتة وغير قابلة للسحب.

بالإضافة لذلك:
- `html, body { touch-action: manipulation }` في ملفين آخرين كان يتعارض مع pan-y.
- زر التحديث `AppUpdatePrompt` كان يستخدم `var(--accent-color, #6366f1)` الافتراضي (أزرق/أخضر النظام)، شريط كامل العرض يغطي زر تسجيل الدخول الذي يقع تحته مباشرة، بدون آلية إخفاء فعّالة.

## ✅ الإصلاحات المُطبَّقة

### 1) إعادة كتابة `src/styles/smooth-touch-experience.css`
- **إزالة `position:fixed` و `overflow:hidden`** من `body` و `#root`.
- إضافة `touch-action: pan-y` للسماح بالسحب العمودي الطبيعي + قفل السحب الجانبي.
- `overscroll-behavior-y: contain` لمنع pull-to-refresh المربك على Chrome Android، مع الحفاظ على السحب الطبيعي.
- `min-height: 100dvh` لاستخدام Dynamic Viewport Height — يحسب شريط المتصفح بشكل ديناميكي.
- `-webkit-overflow-scrolling: touch` لارتداد iOS الناعم.
- ضمان عدم تعطيل اختيار النص في الفقرات والعناوين (للقراءة)، فقط الأزرار غير قابلة للتحديد.

### 2) ملف جديد `src/styles/mobile-scroll-fix.css`
- يُحمَّل **في النهاية** بعد كل ملفات CSS الأخرى ليفوز في cascade.
- يضمن `body { position: relative !important; overflow-y: auto !important }`.
- يصلح `auth-shell` على الموبايل: بدلاً من `grid + place-items:center` (الذي كان يقفل الارتفاع)، أصبح `flex column` يسمح بالتمرير عند تجاوز المحتوى للشاشة.
- يضيف انتقالات صفحات ناعمة (240ms cubic-bezier).
- تأثير لمس بصري `scale(0.97)` للأزرار على الأجهزة باللمس.
- مساحة آمنة سفلية `safe-area-inset-bottom` للأجهزة ذات الـ home indicator.

### 3) إعادة كتابة `src/components/feedback/AppUpdatePrompt.jsx`
- **ألوان البراند البنفسجية** (`#8e3dff → #5b21b6`) بدلاً من الأخضر/الأزرق النظامي.
- **شريط أصغر وأنيق** (toast بحجم 520px max، توسيط على الموبايل، يمين الأسفل على الديسكتوب).
- **زر إغلاق واضح (X)** بدلاً من زر "لاحقاً" المربك.
- **وضع مصغّر**: عند الضغط على X، لا يختفي البانر تماماً بل يتحوّل إلى زر دائري صغير 44×44 في الزاوية اليسرى السفلية مع شارة خضراء نابضة — لا يغطي أي محتوى.
- **هدوء تلقائي لـ 6 ساعات** بعد الإغلاق (cooldown في localStorage)، فلا يُزعج مجدداً.
- `backdrop-filter: blur(14px)` للزجاج المصقول.
- `animation: yamUpdateSlide` للظهور الناعم.
- `safe-area-inset-bottom` احتراماً للشاشات ذات الـ notch.

### 4) إصلاحات `mobile-app-experience.css` و `mobile-first.css`
- استبدال `touch-action: manipulation` على `html, body` بـ `touch-action: pan-y` (يسمح بالسحب الرأسي + يمنع double-tap-zoom).
- تطبيق `user-select: none` فقط على الأزرار والعناصر التفاعلية، وليس على body كله — مما يسمح للمستخدم بنسخ النصوص.

### 5) تنظيف `index.html`
- إزالة الـ `<link>` المباشر لـ `/src/styles/smooth-touch-experience.css` (الذي كان يعمل في dev فقط ويُسبب مشاكل في الإنتاج). الآن الملف يُستورد عبر `main.jsx` ليُعالَج بشكل صحيح بواسطة Vite bundler.

## 🎁 ما الذي ستحصل عليه

| قبل | بعد |
|------|-----|
| ❌ الصفحة "واقفة"، لا سحب لفوق | ✅ سحب طبيعي وسلس بإصبع واحدة |
| ❌ زر تحديث أخضر كامل العرض يغطي زر الدخول | ✅ شريط بنفسجي أنيق متوسّط الحجم، أو زر دائري مصغّر |
| ❌ لا طريقة لإخفاء بانر التحديث | ✅ زر X واضح + 6 ساعات هدوء بعد الإغلاق |
| ❌ تجربة تشعر بأنها صفحة ويب | ✅ ارتداد iOS، انتقالات 240ms، تأثير ضغط على الأزرار |
| ❌ كان يمنع تحديد النص بالكامل | ✅ يمكنك تحديد ونسخ النصوص في الفقرات |

## 🧪 كيفية التحقق

1. افتح الصفحة على الموبايل.
2. اسحب لفوق وتحت — يجب أن تتمرر الصفحة بسلاسة الآن.
3. عند ظهور بانر التحديث:
   - زر "تحديث" بنفسجي واضح، **ليس** هو الزر الكبير الذي يبدو كزر النظام.
   - زر "X" يصغّر البانر إلى نقطة دائرية صغيرة في الزاوية.
   - زر تسجيل الدخول الحقيقي مرئي تماماً ولا يُغطّى.
4. الضغط على الأزرار يُحدث تأثير `scale(0.97)` لحظي — يشعر كتطبيق.

# تقرير تقوية أمان تطبيق الموبايل

تم تنفيذ تحسينات الأمان التالية داخل مشروع الموبايل:

## 1) Crashlytics
- تفعيل Firebase Crashlytics داخل Gradle.
- إضافة طبقة `CrashReporter` لتهيئة التسجيل وربط المستخدم الحالي وتسجيل الأعطال غير المعالجة والأحداث الأمنية.
- ربط تقارير الأعطال بمعلومات الإصدار ونوع البناء وبعض مفاتيح التشخيص.

## 2) Encrypted DB
- تحويل مخزن Signal المحلي إلى قاعدة `signal.secure.db`.
- تشفير القيم المخزنة داخل القاعدة باستخدام AES/GCM ومفتاح محفوظ في Android Keystore عبر `SecureBlobCodec`.
- تفعيل `PRAGMA secure_delete=ON` لتقليل بقايا البيانات الحساسة بعد الحذف.

## 3) Secure media cache
- إضافة `SecureMediaManager` لاستخدام ملفات وسائط مؤقتة داخل `noBackupFilesDir` مع `.nomedia` وتنظيف تلقائي.
- نقل ملفات التسجيلات والرفع المؤقتة إلى مسار داخلي آمن بدل `cacheDir` العام.
- تعطيل Glide disk cache للصور/المعاينات الحساسة لتقليل التخزين على القرص.
- تشديد إعدادات WebView cache وتعطيل file access غير الضروري.

## 4) Session hardening
- إضافة Session ID داخلي وتضمينه في الطلبات عبر `X-Session-Id`.
- إضافة تتبع آخر نشاط للمستخدم، وآخر وقت خروج للخلفية، وعمر الجلسة.
- قفل الجلسة تلقائيًا بعد الخمول أو البقاء في الخلفية لفترة طويلة.
- تحسين استهلاك/تجديد الجلسة وربط Crashlytics بهوية المستخدم بعد تسجيل الدخول.

## 5) Anti automation
- توسيع `SecurityThreatDetector` لاكتشاف:
  - بيئات Emulator غير الموثوقة
  - أدوات Auto Click / Macro / Appium / UIAutomator
  - خدمات Accessibility المشبوهة المرتبطة بالأتمتة
- تسجيل الحوادث الأمنية داخل Crashlytics وإنهاء التطبيق عند اكتشاف تهديد فعلي.

## ملاحظات
- تم الحفاظ على بنية المشروع قدر الإمكان بدون تغيير معماري كبير.
- تم إضافة طبقات أمان قابلة للصيانة داخل مجلد `utils`.

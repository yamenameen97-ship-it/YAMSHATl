# دليل إعادة الهيكلة الشامل لمشروع Yamshat

## مقدمة

تم إعادة هيكلة مشروع Yamshat بالكامل لحل جميع المشاكل المعمارية الرئيسية وتحسين جودة الأكواس والأداء والصيانة.

## المشاكل التي تم حلها

### 1. وجود Frontend قديم وجديد معًا

**المشكلة**: كان هناك ملفات HTML متناثرة في جذر المشروع بالإضافة إلى مجلد Frontend حديث يحتوي على Vite.

**الحل**:
- نقل جميع ملفات HTML القديمة إلى مجلد `frontend/legacy_html` للحفاظ عليها كمرجع
- توحيد جميع الصفحات تحت نظام React Router الموحد
- إزالة مجلد `dist` القديم من المستودع

### 2. ملفات HTML متناثرة

**المشكلة**: 11 ملف HTML منتشرة في جذر المشروع (admin.html, chat.html, feed.html, إلخ).

**الحل**:
- نقل جميع الملفات إلى `frontend/legacy_html`
- إنشاء نظام توجيه موحد في `core/routing/router.js`
- جميع الصفحات الآن تُدار من خلال React Router

### 3. `dist` داخل المشروع

**المشكلة**: مجلد `dist` يحتوي على ملفات البناء المجمعة.

**الحل**:
- حذف مجلد `dist` من المستودع
- إضافة `.gitignore` لاستبعاد `dist` من الإصدارات المستقبلية
- استخدام Vite للبناء والنشر

### 4. تكرار أكواد وCSS وJS

**المشكلة**: ملفات CSS مثل `global.css` و `frontend-global.css` و `responsive.css` و `frontend-responsive.css` متطابقة تقريبًا.

**الحل**:
- دمج جميع ملفات CSS في ملف موحد `styles/index.css`
- حذف الملفات المكررة
- إنشاء نظام CSS منظم مع متغيرات موحدة

### 5. تضارب Routes

**المشكلة**: عدم وجود نظام توجيه مركزي وموحد.

**الحل**:
- إنشاء ملف `core/routing/router.js` يحتوي على جميع المسارات
- تقسيم المسارات إلى: عام (public)، محمي (protected)، وإداري (admin)
- إضافة دوال مساعدة للتحقق من نوع المسار

### 6. ضعف modularization

**المشكلة**: عدم وجود بنية موحدة للمكونات والخدمات.

**الحل**:
- إنشاء بنية معمارية موحدة:
  - `core/`: الخدمات الأساسية
  - `infrastructure/`: الخدمات والـ Hooks
  - `domain/`: منطق الأعمال
  - `presentation/`: المكونات والصفحات

### 7. ضعف state management

**المشكلة**: عدم وجود نظام إدارة حالة موحد.

**الحل**:
- استخدام Zustand لإدارة الحالة
- تقسيم الحالة إلى شرائح منطقية:
  - `authSlice.js`: الحالة المتعلقة بالمصادقة
  - `appSlice.js`: حالة التطبيق العامة
  - `chatSlice.js`: حالة الدردشة
  - `notificationSlice.js`: حالة الإشعارات
  - `uiSlice.js`: حالة واجهة المستخدم

### 8. عدم توحيد API layer

**المشكلة**: عدم وجود عميل HTTP موحد مع معالجة شاملة للأخطاء.

**الحل**:
- إنشاء `core/api/client.js` مع:
  - عميل Axios موحد
  - معالجات الطلبات والاستجابات
  - نظام تخزين مؤقت ذكي
  - استراتيجية إعادة محاولة مع تراجع أسي

### 9. ضعف socket management

**المشكلة**: عدم وجود إدارة موحدة لـ Socket.io.

**الحل**:
- إنشاء `core/socket/manager.js` مع:
  - إدارة الاتصال والقطع
  - قائمة انتظار للأحداث غير المتصلة
  - معالجة تلقائية لإعادة الاتصال
  - منع تسريب الذاكرة

### 10. ضعف upload handling

**المشكلة**: عدم وجود خدمة موحدة للرفع مع تتبع التقدم.

**الحل**:
- إنشاء `infrastructure/services/uploadService.js` مع:
  - رفع الملفات الفردية والمتعددة
  - رفع الملفات الكبيرة على شكل أجزاء
  - تتبع التقدم والإلغاء
  - التحقق من صحة الملفات

### 11. ضعف caching

**المشكلة**: عدم وجود نظام تخزين مؤقت فعال.

**الحل**:
- نظام تخزين مؤقت ذكي في `core/api/client.js`
- إمكانية التحكم في TTL
- إمكانية إعادة التحديث القسري
- تنظيف تلقائي للذاكرة

### 12. ضعف error handling

**المشكلة**: عدم وجود معالجة موحدة للأخطاء.

**الحل**:
- إنشاء `core/utils/errorHandler.js` مع:
  - فئات أخطاء مخصصة
  - معالجة شاملة للأخطاء
  - رسائل خطأ موحدة
  - استراتيجيات التعافي

### 13. احتمال memory leaks

**المشكلة**: عدم وجود آلية لمنع تسريب الذاكرة.

**الحل**:
- تنظيف الموارد في `socketManager`
- إدارة المستمعين (listeners) بشكل صحيح
- تنظيف المؤقتات (timers) والفترات الزمنية
- استخدام WeakMap حيث يكون مناسبًا

### 14. naming غير موحد

**المشكلة**: عدم وجود معايير موحدة لتسمية الملفات والمتغيرات.

**الحل**:
- إنشاء `architecture/NAMING_CONVENTIONS.md`
- معايير موحدة لـ:
  - أسماء الملفات والمجلدات
  - أسماء المتغيرات والثوابت
  - أسماء الدوال والمكونات
  - معرّفات DOM

## الملفات الجديدة الرئيسية

### 1. `core/routing/router.js`
نظام التوجيه الموحد مع دعم المسارات العامة والمحمية والإدارية.

### 2. `core/store/index.js`
متجر Zustand الرئيسي مع جميع الشرائح.

### 3. `core/api/client.js`
عميل HTTP موحد مع التخزين المؤقت وإعادة المحاولة.

### 4. `core/socket/manager.js`
مدير Socket.io الموحد مع معالجة الاتصال والقطع.

### 5. `infrastructure/services/uploadService.js`
خدمة الرفع الموحدة مع دعم الملفات الكبيرة.

### 6. `core/utils/errorHandler.js`
معالج الأخطاء الموحد مع رسائل موحدة.

### 7. `core/utils/logger.js`
نظام السجلات الموحد مع تتبع الأداء.

## معايير التسمية الموحدة

- **المجلدات**: kebab-case (مثل `chat-service`)
- **المكونات**: PascalCase (مثل `ChatComponent.jsx`)
- **الخدمات**: camelCase (مثل `chatService.js`)
- **الـ Hooks**: camelCase مع بادئة `use` (مثل `useAuth.js`)
- **الثوابت**: UPPER_SNAKE_CASE (مثل `API_BASE_URL`)

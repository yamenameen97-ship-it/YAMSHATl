🧠 الميزات الذكية المدمجة (إضافة من عندي)

  • ✓ Snapshot للمحتوى — نلتقط لقطة من المحتوى وقت البلاغ، فحتى لو حذف المخالف محتواه يبقى الدليل محفوظاً.
  • ✓ تجميع البلاغات المتكررة — إذا أبلغ نفس المستخدم مرتين عن نفس الكيان، نزيد العداد بدل إنشاء سجل مكرر.
  • ✓ ترقية الأولوية تلقائياً — البلاغات التي تتجاوز 5 تبليغات تصبح high، و10 تبليغات تصبح urgent.
  • ✓ أولوية حسب السبب — self_harm و violence تصبح urgent فوراً، nudity/hate_speech/impersonation/scam تصبح high.
  • ✓ إشعارات لحظية للمشرفين — كل بلاغ جديد يُنشئ Notification لكل المشرفين.
  • ✓ إشعار المُبلِّغ بنتيجة بلاغه — بعد المعالجة يصل إشعار "✅ تمت معالجة بلاغك".
  • ✓ سجل أحداث كامل (Audit Trail) — جدول report_events يحفظ كل تغيير على البلاغ.
  • ✓ منع البلاغ على النفس — التحقق من أن target_id != current_user.id.
  • ✓ تسجيل IP و User-Agent — لمكافحة سوء استخدام نظام البلاغات نفسه.
  • ✓ إجراءات جماعية (Bulk Actions) — معالجة عدة بلاغات بنقرة واحدة من لوحة الأدمن.

🧪 اختبار سريع

# 1) إنشاء بلاغ (من المستخدم)
curl -X POST https://api.yamshat.com/api/reports \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "target_type": "post",
    "target_id": "123",
    "reason": "spam",
    "details": "منشور إعلاني مكرر"
  }'

# 2) عرض البلاغات (من الأدمن)
curl https://api.yamshat.com/api/reports/admin?status=pending \
  -H "Authorization: Bearer $ADMIN_TOKEN"

# 3) اتخاذ إجراء
curl -X POST https://api.yamshat.com/api/reports/admin/1/action \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -d '{"action":"remove_content","notes":"محتوى دعائي"}'

🔔 الإشعارات المدمجة

أنواع الإشعارات الجديدة في جدول notifications:

  • REPORT_NEW — تصل للمشرفين عند إنشاء بلاغ جديد
  • REPORT_RESOLVED — تصل للمُبلِّغ عند معالجة بلاغه
  • MODERATION_WARNING — تحذير من فريق الإشراف
  • MODERATION_MUTE — إشعار بكتم الحساب مؤقتاً

# تقرير إصلاحات v53

## 🛠️ المشاكل المُصلحة

### 1. فشل تسجيل الدخول مع Internal Server Error (500)

**السبب الجذري:**
- معالج الأخطاء العام في `backend/app/main.py` كان يبتلع كل الأخطاء ويعيد 500
  حتى عندما يكون السبب الحقيقي 400 (كابتشا خاطئة) أو 401 (كلمة مرور خاطئة) أو 429 (Rate limit) ...إلخ.
- بعض الأخطاء العابرة في خدمات الـ Redis أو قاعدة البيانات أو إرسال البريد كانت تُسقط تسجيل الدخول كاملاً.

**الإصلاح:**
- **`app/main.py`**:
  - أُضيف معالج مخصّص لـ `StarletteHTTPException` يحافظ على الـ status code الأصلي (400/401/403/429/...) ويُلحق ترويسات CORS.
  - المعالج العام لـ `Exception` صار يميّز HTTPException ويُمرّرها للمعالج الصحيح بدل تحويلها إلى 500.
  - الاستجابة الجديدة تُضيف `error_type` لتسهيل التشخيص اللاحق.

- **`app/api/routes/auth.py`** — مسار `POST /api/auth/login`:
  - الدالة الرئيسية أصبحت غلافاً يستدعي `_login_impl` ويلتقط أي خطأ غير متوقع ويحوّله إلى **503** برسالة واضحة بدلاً من 500 غامض.
  - فحوصات Rate Limit (Redis) و `mark_successful_login` و `is_suspicious_login` و commit ما بعد الدخول صارت كلها داخل try/except لا تُسقط الطلب عند فشل عابر.
  - إصدار رمز التحقق وإرسال البريد (في حالة 403 verification_required) أصبح هو الآخر محمياً، فلا يفشل الطلب لو فشل SMTP.
  - HTTPException الأصلية تمرّ كما هي للمعالج الجديد، فيراها المستخدم بالسبب الحقيقي (مثلاً: "كلمة المرور غير صحيحة" بدل "Internal server error").

النتيجة: المستخدم يرى الآن رسالة دقيقة (كابتشا خاطئة / كلمة مرور غير صحيحة / محاولات كثيرة) بدلاً من شاشة الخطأ الحمراء.

### 2. شريط "إصدار جديد متاح" الأخضر العريض

**المشكلة:**
- الإشعار كان يظهر بشريط أخضر/بنفسجي عريض بأسفل الشاشة (شبيه بإشعار "تنشيط Windows") ويغطي الأزرار.

**الإصلاح:**
- **`frontend/src/components/feedback/AppUpdatePrompt.jsx`** أُعيدت كتابته بالكامل:
  - تم **حذف الشريط الأخضر/البنفسجي السفلي نهائياً**.
  - الإشعار الآن **زر صغير دائري في الزاوية العلوية اليسرى** بنفس استايل نظام إشعارات التطبيق (دائرة داكنة + شارة نقطة بنفسجية نابضة) — مطابق تماماً لاستايل بقية الإشعارات داخل التطبيق.
  - الضغط على الزر يطبّق التحديث فوراً (`SKIP_WAITING` + إعادة تحميل).
  - زر إغلاق صغير بجواره يُخفي الإشعار لمدة 12 ساعة.
  - يحترم `safe-area-inset-top` على أجهزة iOS والوضع PWA standalone.

النتيجة: لا مزيد من الشريط الأخضر — فقط أيقونة صغيرة أنيقة بالزاوية العلوية كما طُلب.

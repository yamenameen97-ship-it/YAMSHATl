# إصلاحات الرسائل الصوتية وتكسر ملفات الوسائط — تقرير فني

## ✅ المشاكل التي تم حلها

### 1) فشل إرسال الصوت + الصوت يأخذ مساحة كبيرة في الدردشة
كان عرض الرسالة الصوتية يستخدم بطاقة كاملة العرض داخل فقاعة الرسالة، تحتوي على:
- عنوان "رسالة صوتية" وزمن طويل
- **4 أزرار سرعة دائمة الظهور** (×1, ×1.25, ×1.5, ×2)
- صف منفصل لموجة الصوت

النتيجة: شريط صوتي ضخم يستهلك ارتفاع الشاشة.

### 2) "تكسر الملفات" عند إرسال صور أو فيديوهات (السبب الجذري)
الـ Backend كان يستخدم `werkzeug.secure_filename()` لتنظيف أسماء الملفات. هذه الدالة **تحذف كل الأحرف غير ASCII**، فإذا رفع المستخدم ملفاً اسمه عربي:

| اسم الملف الأصلي | بعد `secure_filename` | المشكلة |
|---|---|---|
| `صورة.jpg` | `jpg` (بدون نقطة) | لا يوجد امتداد → ملف مكسور |
| `فيديو.mp4` | `mp4` (بدون نقطة) | لا يوجد امتداد → ملف مكسور |
| `تسجيل.ogg` | `ogg` (بدون نقطة) | لا يوجد امتداد → ملف مكسور |

عند الحفظ، اسم الملف يصبح `abc123_jpg` (بدون نقطة وامتداد)، والمتصفح يفشل في تحديد نوع الملف عند العرض → الصورة/الفيديو يظهر مكسوراً.

### 3) مشكلة كودك MIME الإضافية
`MediaRecorder` ينتج رسائل صوتية بنوع MIME يحتوي على معامل الكودك:
```
audio/webm;codecs=opus
```
عند فحص هذا النوع ضد قائمة بيضاء صريحة (`VITE_MEDIA_ALLOWED_TYPES=audio/webm,...`)، **لا يطابق** أي شرط لأن النص الكامل به `;codecs=opus`. النتيجة: رفض الصوت قبل وصوله للسيرفر.

## 🛠️ الملفات المُعدَّلة

### Frontend

#### `frontend/src/components/ui/VoiceMessagePlayer.jsx` (مُعاد كتابته)
- إضافة وضع `bubbleless` لعرض المشغل كـ pill أفقي صغير (نمط واتساب)
- أيقونات SVG (▶ / ⏸) بدل النصوص
- زر سرعة واحد فقط يظهر بعد بدء التشغيل (دورة ×1 → ×1.5 → ×2)
- معالجة أخطاء التحميل
- استماع لحدث `durationchange` (مهم لملفات الصوت من Blob حيث `loadedmetadata` لا تعطي مدة فورية في Chrome/Edge)

#### `frontend/src/components/chat/MessageBubble.jsx`
- اكتشاف الرسائل الصوتية الخالصة (`isVoiceOnly`) وعرضها بدون فقاعة محيطة
- تمرير `bubbleless` و `isMe` للمشغل
- إضافة class `is-voice-only` و `voice-only` للتحكم بالأنماط

#### `frontend/src/components/chat/VoiceRecorder.jsx`
- إضافة `normalizeMime()` لتجريد معامل الكودك (`audio/webm;codecs=opus` → `audio/webm`)
- إضافة `extensionForMime()` لاختيار الامتداد الصحيح حسب MIME الأساسي
- توسيع `CODEC_PRIORITY` ليشمل `mp4`, `mpeg`, `ogg` للأجهزة المختلفة
- ضمان أن الملف المُرسل يحمل MIME أساسياً نظيفاً وامتداداً صحيحاً

#### `frontend/src/services/media/mediaUploadService.js`
- إضافة `baseMime()` لتجريد المعاملات
- تحديث `validateFile()` لمقارنة الـ base MIME وليس النص الكامل، مع سماح ذكي لكل `audio/*` إذا كانت القائمة تحتوي على نوع صوتي
- تحديث `uploadVoiceNote()` لتطبيع MIME والاسم قبل إنشاء `File`

#### `frontend/src/components/chat/ChatInput.jsx`
- استبدال `<audio controls>` الأصلي للمتصفح (الضخم جداً) بـ `VoiceMessagePlayer bubbleless` لمعاينة الصوت قبل الإرسال

#### `frontend/src/styles/chat-premium.css`
- **أنماط `.yam-voice-pill`**: حاوية أفقية مدمجة بعرض 200-320px، ارتفاع تلقائي صغير
- **`.yam-bubble.is-voice-only`**: إزالة الخلفية والحدود والـ padding عندما تحتوي الفقاعة على صوت فقط
- استجابة كاملة للجوال (`@media max-width: 480px`)
- ألوان مختلفة للمرسل/المستقبل (gradient للمرسل، خلفية شفافة للمستقبل)

### Backend

#### `backend/app/api/routes/upload.py` (الإصلاح الجذري)
- إضافة `_MIME_EXTENSION_MAP` خريطة صريحة لـ 23 نوع MIME شائعاً
- إضافة `_base_mime()`: تجريد المعاملات من content-type
- إضافة `_extension_from_mime()`: استنتاج الامتداد من MIME (مع fallback إلى `mimetypes.guess_extension`)
- إضافة `_split_extension()`: استخراج آمن للجذر والامتداد
- إضافة **`_safe_storage_name()`**: الإصلاح الجوهري — يضمن أن أي ملف (حتى الأسماء العربية البحتة) يُحفظ بامتداد صحيح:
  1. يحاول `secure_filename` أولاً
  2. إذا فقد الامتداد، يأخذه من الاسم الأصلي
  3. إذا لم يجد، يستنتجه من content-type
  4. يقصر الجذر لـ 80 حرفاً (تجنب أسماء طويلة)
- استبدال جميع استدعاءات `secure_filename` المباشرة بـ `_safe_storage_name()` في:
  - `save_upload()`
  - `init_resumable_upload()`
  - `_session_file_path()` (مع تمرير content-type)
  - `complete_resumable_upload()`

## 🧪 سلوك متوقع بعد الإصلاح

| السيناريو | قبل | بعد |
|---|---|---|
| رفع `صورة.jpg` | يُحفظ كـ `abc_jpg` → مكسور | يُحفظ كـ `abc_upload_xxx.jpg` → يعمل |
| رفع `فيديو.mp4` | يُحفظ كـ `abc_mp4` → مكسور | يُحفظ كـ `abc_upload_xxx.mp4` → يعمل |
| إرسال صوت (Chrome) | رُفض بسبب `audio/webm;codecs=opus` | يُقبل كـ `audio/webm` ويُحفظ `.webm` |
| إرسال صوت (Safari) | قد يفشل بسبب `audio/mp4` | يُحفظ كـ `.m4a` ويعمل |
| عرض رسالة صوتية | بطاقة كاملة بـ 4 أزرار سرعة وحدود | شريط أفقي صغير 200-320px مثل واتساب |
| معاينة الصوت في الإدخال | عناصر تحكم متصفح ضخمة | شريط صغير متناسق مع التصميم |

## 🔄 لا توجد تغييرات في:
- مخطط قاعدة البيانات (Message.media_url لا يزال يحفظ المسار/URL كما هو)
- بروتوكول API (`POST /send_message`, `POST /upload`)
- منطق WebSocket
- خدمة Cloudinary (تعمل تلقائياً مع الأسماء الجديدة الصحيحة)

لا حاجة لـ migration أو إعادة نشر متعدد المراحل.

# v77 — إعدادات مضغوطة على صفحة واحدة

## المشكلة السابقة
- الأزرار في صفحة الإعدادات كبيرة الحجم (ارتفاع ~40px، padding واسع).
- عدة تبويبات كانت تنقل المستخدم لصفحات فرعية (`/settings/feed`, `/settings/reels`, ...) بدلاً من عرض الإعداد فوراً.
- الخط الكبير + الفراغات = المحتوى يمتد على عدة شاشات مما يعيق الوصول السريع.

## الحل (v77)
### 1) في `frontend/src/pages/Settings.jsx` (إعادة كتابة كاملة)
- **إزالة روابط الخروج**: لم تعد التبويبات تُخرج المستخدم من الصفحة. كل تبويب (feed / reels / stories / inbox / voice / engagement / wallet / profile / notifications) يعرض إعداداته الأساسية مباشرة داخل نفس الصفحة.
- **زر اختياري "فتح الصفحة الكاملة ›"** لكل قسم له صفحة فرعية متقدمة — لمن يريد الإعدادات التفصيلية.
- **زر مصغّر موحّد `settings-btn-mini`** — ارتفاع 24px بدلاً من 40+px.
- **تصغير**: 
  - font-size رئيسي: 15px → 12.5px
  - padding الصفوف: 14px → 7px
  - padding البطاقات: 18px → 12px
  - Toggle: 50×28 → 36×20
  - Selects: 40px height → 26px height
- **CSS override قوي** يجبر كل `.btn / .btn-small / .btn-medium` داخل `.settings-main` على ارتفاع 24px.

### 2) في `frontend/src/components/settings/SettingsShell.jsx`
- تطبيق نفس تصغير padding / font-size / أزرار / toggles للصفحات الفرعية حتى تكون متناسقة عند فتحها.
- زر الرجوع 44×44 → 32×32.

## النتيجة
- كل الإعدادات الرئيسية معروضة في الصفحة نفسها.
- إمكانية عرض 15–20 إعدادًا في viewport واحد بدل 5–6.
- الأزرار مصغّرة بشكل موحّد بأصغر حجم قابل للاستعمال (24px height).
- لا فراغات كبيرة بين الصفوف.
- الشريط الجانبي أصبح أضيق (200px) لإفساح مساحة للمحتوى.
- Responsive: على الشاشات الصغيرة يتحول الشريط الجانبي لعمودين ثم عمود واحد.

## ملفات معدّلة

# إصلاح v88.5 — استطلاعات المنشورات (Poll) بأزرار تصويت ونسب

## المشكلة
عند نشر استطلاع من مؤلّف المنشورات في صفحة المنشورات (Feed):
- كان يظهر **سؤال الاستطلاع فقط كنص** ضمن المنشور.
- **أزرار الاختيار لم تكن تظهر**، فلم يستطع الجمهور التصويت.
- بينما نفس الميزة كانت تعمل بشكل صحيح في استطلاعات المجموعات.

## السبب الجذري (Root Cause)
1. **الـ backend كان يعيد `poll` بشكل سليم** ضمن كل منشور (id, label, votes, voted_by_me).
2. **لكن الواجهة كانت تُهمل هذا الحقل** في مرحلتين:
   - `pages/FeedMobile.jsx` → دالة `mapBackendPost` لم تكن تنسخ `poll` إلى كائن المنشور.
   - `components/mobile/MobilePostCard.jsx` لم يكن يعرض أي UI للاستطلاعات إطلاقاً.
   - `pages/FeedEnhanced.jsx` (نسخة الديسكتوب) — نفس المشكلة في mapper وفي `PostCard` المحلي.

## الإصلاحات المُطبَّقة

### 1) FeedMobile.jsx — تمرير `poll` من backend إلى بطاقة المنشور
```js
// في mapBackendPost:
poll: Array.isArray(p.poll) ? p.poll : (Array.isArray(p.poll_options) ? p.poll_options : []),
```

### 2) MobilePostCard.jsx — إضافة UI الاستطلاع كاملاً
- استيراد `votePoll` من `api/posts.js`.
- حالة محلية `pollItems` + `pollBusy` مع تحديث متفائل + تراجع عند فشل الشبكة.
- عرض شريط تعبئة (fill bar) + النسبة المئوية + علامة صوّتك لكل خيار.
- تعطيل الأزرار بعد التصويت.
- دعم كامل لـ RTL (شريط النسبة يبدأ من اليمين).

### 3) FeedEnhanced.jsx — نفس المعالجة للديسكتوب
- إضافة `poll` إلى mapper.
- استيراد `votePoll` وإضافة الحالة والمعالج في `PostCard` المحلي.
- إدراج نفس شكل UI الاستطلاع بين نص المنشور والوسائط.

### 4) CSS — feed-actions-compact-v39.css
تنسيقات `.yam-post-poll` (متطابقة تماماً بصرياً مع تنسيقات استطلاعات المجموعات: خلفية بنفسجية شفافة، حواف دائرية، شريط تعبئة متدرج، علامة صح على الخيار المُختار).

## الملفات المُعدَّلة
- `frontend/src/pages/FeedMobile.jsx`
- `frontend/src/pages/FeedEnhanced.jsx`
- `frontend/src/components/mobile/MobilePostCard.jsx`
- `frontend/src/styles/feed-actions-compact-v39.css`

## النتيجة المتوقعة
عند نشر استطلاع في المنشورات الآن:
1. يظهر شكل الاستطلاع مطابقاً لاستطلاعات المجموعات (نفس النمط والألوان).
2. تظهر الأزرار لكل خيار (`الخيار 1`, `الخيار 2`, ...).
3. عند الضغط: يُسجَّل الصوت عبر `POST /posts/{id}/poll-vote` ثم تتحدّث النسب فوراً.
4. بعد التصويت: تُقفَل الأزرار وتظهر النسبة المئوية بجانب كل خيار + علامة صح على الخيار المختار.
5. عدد الأصوات الإجمالي يظهر أعلى الاستطلاع.

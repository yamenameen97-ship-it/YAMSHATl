# v88.6 — إصلاح جذري لنظام الإشعارات

## المشكلة كما وصفها المستخدم
- الأيقونة في أعلى الهيدر تعرض رقماً (مثلاً «6»).
- عند فتح صفحة `/notifications` لا تظهر أي إشعارات على الإطلاق (الصفحة فارغة).

## السبب الجذري (Root Cause)

### 1) الصفحة تعرض «فارغة» رغم أن الأيقونة تعرض رقماً

في `src/pages/Notifications.jsx` كانت القائمة تعتمد على مكتبة `react-window`
عبر `FixedSizeList` مع `AutoSizer` من `react-virtualized-auto-sizer`:

```jsx
<div style={{ height: 'calc(100vh - 70px)', display: 'flex', ... }}>
  ...
  <div style={{ flex: 1 }}>
    <AutoSizer>
      {({ height, width }) => (
        <List height={height} width={width} itemCount={...} itemSize={160}>
          {NotificationRow}
        </List>
      )}
    </AutoSizer>
  </div>
</div>
```

بعد إصلاح v86.4 (لتفعيل السحب على الجوال)، صار الجذر
`.yam-notifications-page` يفرض `height: auto` على شاشات ≤ 900px
والحاوية الداخلية `flex:1` تحوّلت إلى `flex: 0 0 auto` مع `min-height: 60vh`.

النتيجة: `AutoSizer` يحتاج لأن يقيس ارتفاعاً عددياً محسوباً على والده
المباشر (ResizeObserver). لكن لأن سلسلة الآباء صارت `height: auto`،
ثم `AutoSizer` نفسه يبني عنصراً بارتفاع 0 افتراضياً، فإنه يبلغ
`height: 0` لـ `FixedSizeList` → عدد الصفوف المُصيَّرة = 0 → **الصفحة تظهر فارغة تماماً**
حتى لو كانت `filteredItems.length === 6`.

على سطح المكتب المشكلة أخفّ لكنها كانت تظهر أيضاً حين يكون هناك
عنصر آخر يزيد على ارتفاع `calc(100vh - 70px)` (كارت الرأس + شريط الفلاتر).

### 2) الأيقونة والصفحة يقرآن من مصدرين مختلفين

- `Topbar.jsx` يعرض العدّاد اعتماداً على **React Query** فقط
  (`['topbar-notifications-count']`) — يستدعي `getNotifications(20)`
  كل 20 ثانية.
- `Notifications.jsx` يقرأ من **Zustand store** (`useNotificationStore`).
- عند تعليم إشعار كمقروء داخل الصفحة، كان الكود يعدّل الـ store
  **فقط** — بدون:
  - استدعاء API الفعلي `POST /notifications/{id}/read` (فيرجع الإشعار
    غير مقروء بعد إعادة التحميل — bug صامت).
  - إبطال React Query cache — فتبقى الشارة تعرض الرقم القديم
    لعشرات الثواني.

### 3) الاستقبال في الوقت الحقيقي كان يحدّث الـ store فقط

عند وصول إشعار جديد عبر WebSocket:
- `GlobalNotificationListener` يحدّث الـ store ✅.
- **لا** يبطل كاش React Query الخاص بـ Topbar ❌.
- الشارة قد تتأخر حتى الـ polling التالي (حتى 20 ثانية).

### 4) طلب إذن إشعارات المتصفح كان يفشل صامتاً

كان الجذر يعتمد على `Notification.permission` أثناء mount ولا يوجد أي
مسار واضح لإعادة الطلب من user gesture لاحقاً.

## الإصلاحات المطبّقة

### ملف `src/pages/Notifications.jsx` (إعادة كتابة كاملة)

1. **إزالة `react-window` و `AutoSizer` كلياً.** الحد الأعلى للإشعارات
   المعروضة = 50، لا حاجة إطلاقاً للـ virtualization، وقد كان هو المسبب
   المباشر لفراغ الصفحة على الجوال.
2. **قائمة بسيطة بـ `map` عادي** — مضمون العرض على كل الأحجام،
   وتعمل مع كل الإصلاحات السابقة للسحب على الجوال دون تعارض.
3. **تخطيط جذر مرن**: `min-height: calc(100dvh - 70px)` بدل ارتفاع
   ثابت. يعمل على الجوال (100dvh = ارتفاع فعلي حقيقي بعد شريط العنوان)
   وعلى السطح دون قيود.
4. **استعادة فورية من localStorage**: نستدعي `restoreFromStorage()`
   قبل الشبكة، فتظهر الإشعارات المخزّنة محلياً فوراً دون فراغ ظاهر.
5. **حماية من مسح المخزن على استجابة فارغة**: إذا رجعت الشبكة بمصفوفة
   فارغة (مثلاً 401 مؤقّت)، لا نمسح الإشعارات المستعادة — يظل المستخدم
   يرى ما لديه ثم يُعاد التحميل عند visibilitychange.
6. **`markRead` يستدعي API الفعلي**: تحديث تفاؤلي فوري في الـ store +
   `POST /notifications/{id}/read` — فتبقى العلامة بعد إعادة التحميل.
7. **`markAllRead` يرسل IDs محدّدة**: يستدعي `markNotificationsRead(unreadIds)`
   بدل تعليم كل شيء أعمى.
8. **حدث عام عند التغيير**: `window.dispatchEvent('yamshat:notifications-changed')`
   بعد كل عملية قراءة/إضافة، لكي يبطل الـ Topbar كاشه فوراً.
9. **بانر «فعّل الإشعارات»** يظهر إذا كان الإذن `default` أو `denied`،
   ويفتح مربع الإذن داخل نفس click gesture.
10. **إعادة تحميل عند العودة إلى التبويب** (`visibilitychange`).
11. **رسائل حالة أوضح**: تحميل، خطأ + زر إعادة، لا يوجد إشعارات
    (مع تمييز بين «الكل فارغ» و«لا يوجد غير مقروء»).

### ملف `src/components/layout/Topbar.jsx` (إضافات)

1. **قراءة مزدوجة للعدّاد**: من React Query (سيرفر) ومن Zustand store
   (بعد التحديثات التفاؤلية)، ونأخذ الأصغر — فالشارة تعكس آخر حالة فعلية.
2. **مستمع `yamshat:notifications-changed`**: يبطل الكاش فوراً بعد
   أي تغيير (قراءة، مسح، ...).
3. **مستمع سوكت `new_notification`**: يبطل الكاش فوراً + يدفع للـ store،
   فتظهر الشارة بلا انتظار polling.

## اختبار

- افتح `/notifications` بعد استقبال إشعارات — يجب أن ترى القائمة مباشرة.
- علّم إشعاراً كمقروء — الشارة في الجرس تنقص فوراً.
- تحديث الصفحة — الحالة تبقى محفوظة (على الخادم).
- عبر الجوال (≤ 900px) — القائمة تظهر ويمكن السحب.
- إذا كان الإذن `default` — يظهر بانر يطلب التفعيل داخل click.

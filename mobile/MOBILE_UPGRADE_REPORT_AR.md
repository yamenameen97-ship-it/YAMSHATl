# تقرير التحسينات والترقيعات - Yamshat Mobile v2.4.0

## تم إضافته فعلياً في هذه النسخة

### 1) الأمان والجلسة والتوكنات
- تخزين الجلسة بشكل مشفّر باستخدام `EncryptedSharedPreferences` بدل التخزين العادي.
- إضافة `refresh token` وإدارة انتهاء صلاحية `access token` و `refresh token`.
- تجديد تلقائي للجلسة عند استلام 401 عبر `RefreshTokenAuthenticator`.
- تسجيل خروج تلقائي إذا انتهت الجلسة أو فشل التجديد.
- حماية أوضح لطلبات الـ API عبر `AuthInterceptor` + `RetryInterceptor`.
- تفعيل `FLAG_SECURE` على الشاشات الحساسة لتقليل تسريب لقطات الشاشة.
- تحسين قواعد الـ ProGuard/obfuscation.
- إضافة Deep Linking من روابط الويب ومن مخطط `yamshat://open/...`.

### 2) مقاومة السبام وسوء الاستخدام داخل الموبايل
- `ActionRateLimiter` لمنع الضغطات السريعة جداً في:
  - تسجيل الدخول
  - إنشاء الحساب
  - تأكيد البريد وإعادة الإرسال
  - إرسال الرسائل الخاصة
  - إرسال رسائل المجموعات
  - رفع/إرسال الميديا
- رسائل واضحة للمستخدم عند تكرار المحاولات بسرعة.

### 3) الاتصال والاعتمادية
- `NetworkMonitor` لاكتشاف الاتصال بالإنترنت لحظياً.
- بانر Offline داخل الواجهة الرئيسية.
- إعادة محاولة تلقائية للطلبات الفاشلة القابلة للإعادة (Retry) مع Backoff.
- WebView يعرض شاشة Offline بدلاً من الفشل الصامت.
- حفظ آخر محادثات خاصة/جماعية محلياً وعرضها عند فشل الشبكة.

### 4) الشات
- حفظ آخر الرسائل محلياً `ConversationCache` للمحادثات الخاصة والمجموعات.
- Optimistic UI مبدئي للرسائل الخاصة والمجموعات.
- حالات أوضح للرسائل: جاري الإرسال / بانتظار الشبكة / فشل / تم التسليم / تمت المشاهدة.
- تقليل السبام في إرسال النصوص والميديا.

### 5) الإشعارات
- عدّاد Badge محلي للإشعارات غير المقروءة.
- مزامنة أفضل لـ FCM token مع حفظ pending token لو المستخدم لم يسجل دخول بعد.
- تصفير العدّاد عند فتح شاشة الإشعارات.

### 6) الـ UX
- Splash Screen أفضل بصرياً.
- إضافة Onboarding من 3 خطوات لأول تشغيل.
- إصلاح/تنظيف `MainActivity` والـ navigation الداخلية.
- دعم Offline banner داخل الشاشة الرئيسية.

## خدمات كانت موجودة بالفعل وتم الحفاظ عليها
- Firebase Messaging + Firebase Analytics.
- LiveKit / البث والمكالمات.
- WebSocket reconnect الأساسي.
- فصل مبدئي بين user / admin.
- Auth Guard أساسي على الشاشات الرئيسية.
- دعم الميديا في الشات.
- typing indicator و seen/delivered.
- الوضع الداكن والهوية البصرية الداكنة.

## ما زال يحتاج تطوير إضافي لاحقاً لو عايز نسخة أقوى
- Offline sync حقيقي للرسائل المعلّقة مع Queue كاملة تُرسل تلقائياً بعد رجوع الإنترنت.
- Pagination كاملة للرسائل القديمة الخاصة.
- لوحة Admin Native كاملة داخل تطبيق الأندرويد نفسه.
- Crash reporting احترافي مثل Firebase Crashlytics.
- Feature flags و OTA updates.
- SSL pinning فعلي بشهادات الإنتاج النهائية.
- حماية tampering / root / hook detection بشكل أوسع.
- State management أوسع لو سيتم تقليل الاعتماد على WebView مستقبلاً.
